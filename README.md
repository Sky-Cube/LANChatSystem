# 基于java的局域网聊天系统 (LANChatSystem)

一个基于Java Socket的局域网即时通讯系统，支持群聊和私聊功能。

## 项目简介

本项目是一个简单的局域网聊天系统，采用客户端-服务器架构，使用Java Socket通信实现局域网内的即时消息传递。系统提供友好的图形用户界面，支持用户登录、在线用户列表显示、群聊和私聊等功能。
后续将更新使用Web实现客户端，实现网页聊天系统并接入AI智能助手，也将继续完善其他功能.

### 核心模块说明

#### 服务器端 (LanChatServer)

**1. Server.java - 主服务器类**
- 功能：服务器启动入口，负责监听客户端连接
- 核心组件：
  - `ServerSocket` - 监听8888端口
  - `Map<Socket, String> onLineSocket` - 存储在线用户的Socket和昵称映射
  - 循环等待客户端连接，为每个客户端创建独立处理线程

**2. ServerReaderThread.java - 消息处理线程**
- 功能：处理单个客户端的所有消息
- 支持三种消息类型：
  - **类型1** - 用户登录：更新在线用户列表并广播
  - **类型2** - 群聊消息：转发给所有在线用户
  - **类型3** - 私聊消息：转发给指定用户
- 核心方法：
  - `updateClientOnLineList()` - 更新所有客户端的在线列表
  - `sendMsgToAll()` - 群聊消息广播
  - `sendPrivateMsg()` - 私聊消息定向转发

**3. Constant.java - 常量定义**
- 服务器IP地址（默认：127.0.0.1）
- 服务器端口号（默认：8888）

#### 客户端 (LanChatClient)

**1. AppStart.java - 客户端启动类**
- 程序入口，启动登录界面

**2. LoginFrame.java - 登录界面**
- 功能：用户输入昵称，连接服务器
- UI组件：
  - 昵称输入框（带圆角设计）
  - 登录按钮（渐变背景）
  - 退出按钮
- 验证逻辑：检查昵称非空且不重复

**3. ClientChatFrame.java - 聊天主界面**
- 功能：显示聊天消息、输入消息、显示在线用户
- UI布局：
  - 左侧：聊天消息显示区（JTextPane）
  - 右下：消息输入区（JTextArea）+ 发送按钮
  - 右侧：在线用户列表（JList）+ 私聊按钮
- 核心功能：
  - 区分显示群聊和私聊消息
  - 实时更新在线用户列表
  - 显示消息时间戳

**4. ClientReaderThread.java - 消息接收线程**
- 功能：持续监听服务器消息并更新界面
- 消息处理：
  - 类型1：更新在线用户列表
  - 类型2：显示群聊消息
  - 类型3：显示私聊消息

**5. Constant.java - 客户端常量**
- 服务器连接配置（IP和端口）

### 通信协议

#### 消息格式定义

所有消息通过Java的DataInputStream和DataOutputStream传输，采用二进制协议：

**1. 用户登录消息**
```
┌───────────┬────────────────┐
│ 类型(1)   │  昵称(UTF-8)  │
│  (int)    │   (String)    │
└───────────┴────────────────┘
```

**2. 群聊消息**
```
┌───────────┬────────────────┐
│ 类型(2)   │ 消息内容(UTF) │
│  (int)    │   (String)    │
└───────────┴────────────────┘
```

**3. 私聊消息**
```
┌───────────┬──────────┬──────────┬──────────┐
│ 类型(3)   │ 发送者   │ 接收者   │ 消息内容 │
│  (int)    │ (String) │ (String) │ (String) │
└───────────┴──────────┴──────────┴──────────┘
```

**4. 在线列表更新**
```
┌───────────┬──────────┬────────────┬────────────┐
│ 类型(1)   │ 用户数量 │  用户1    │  用户2...  │
│  (int)    │  (int)   │  (String) │  (String)  │
└───────────┴──────────┴────────────┴────────────┘
```

## 快速开始

### 环境要求

- JDK 8 或更高版本
- IntelliJ IDEA（推荐）或 Eclipse
- Windows/Linux/macOS 系统

### 运行步骤

#### 第一步：启动服务器

1. 在 IntelliJ IDEA 中打开项目
2. 找到文件：`LanChatServer/src/project/chat/server/Server.java`
3. 右键点击，选择 "Run 'Server.main()'"
4. 控制台显示：
   ```
   启动服务器端
   等待客户端连接...
   当前本机时间：xxxx-xx-xx xx:xx:xx
   ```

#### 第二步：启动客户端

1. 找到文件：`LanChatClient/src/AppStart.java`
2. 右键点击，选择 "Run 'AppStart.main()'"
3. 弹出登录界面
4. 可以启动多个客户端进行测试

#### 第三步：使用系统

**登录：**
- 在登录界面输入昵称
- 点击"登录"按钮

**群聊：**
- 在下方输入框输入消息
- 点击"发送"按钮
- 消息会发送给所有在线用户

**私聊：**
- 在右侧用户列表中选择要私聊的用户
- 点击"私聊"按钮
- 输入消息并发送，只有指定用户能看到

## 配置说明

### 修改服务器IP和端口

如果需要在真实局域网中使用，需要修改服务器IP：

**服务器端配置：**
文件：`LanChatServer/src/project/chat/server/Constant.java`
```java
public class Constant {
    public static final String Server_ip = "192.168.1.100";  // 改为实际服务器IP
    public static final int Server_port = 8888;
}
```

**客户端配置：**
文件：`LanChatClient/src/project/chat/ui/Constant.java`
```java
public class Constant {
    public static final String Server_ip = "192.168.1.100";  // 与服务器IP一致
    public static final int Server_port = 8888;
}
```

### 端口冲突处理

如果8888端口被占用，修改 `Server_port` 为其他端口（如：9999）

## 常见问题

### Q: 客户端无法连接服务器？
A: 检查以下几点：
1. 服务器是否已启动
2. 服务器IP和端口是否配置正确
3. 防火墙是否阻止了连接
4. 客户端和服务器是否在同一局域网

### Q: 中文显示乱码？
A: 确保使用UTF-8编码，项目编码设置为UTF-8

### Q: 如何在局域网中使用？
A: 
1. 将服务器部署在局域网内一台电脑上
2. 修改服务器IP为该机的实际IP
3. 所有客户端都连接到该IP

### Q: 可以同时运行多少个客户端？
A: 理论上没有限制，取决于服务器性能
